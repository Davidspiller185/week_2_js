1. ההבדל בין main develop main - מכיל קוד מוכן develop - זה ענף פיתוח ששם יש את כל הפיצ'רים החדשים לפני הmain
2. feataure branch נוצר מdevelop ולא מתוך main
3. בפרויקט קטן מספיק להשתמש בmain עם feature branch
4. develop משמש כענף שבו יש את כל השינויים החדשים לפני שהם מועברים לmain
5. git workflow נותןד דרך מסודרת כך שכל אחד יכול לעבוד לפתח בענף משלו הלי להפריע לאחרים ועוזר לצמצם merge conflict ולוודא שרק קוד תקין ושנבדק יגיע לmain
6. לשמור בצד שינויים לא גמורים בלי לבצע commit git stash זה שומר את כל השינויים שלא בוצע commit 
7. הפקודה כדי להחזיר את השינויים git stash pop זה מחזיר את השינויים השמורים ומוחקת אותם מרשימת הstash
8. git stash - שומר את השינויים באופן זמני ולא יוצר commit זה רק נועד אם רוצים לעבור משימה ולחזור אחר כך Commit - שומר את השינויים באופן קבוע בהיסטוריית git מיועד שהעבודה נמצאת במצב שצריך לשמור.
9. בלי Git stach אם היינו עוברים לmain המעבר היה גורם לאיבוד השינויים המקומים ולכן git עוצר את הפעולה עם git stash .
10. git revert <comit-hash> יוצר commit חדש שמבטל את השינויים שהוא הכניס 
11. git revet - יוצר Commit חדש שמבטל את השינויים ולא משנה את היסטוריית הCommit git reset - מחזיר את הbranch לcommit קודם ומשנה את ההיסטוריה של Commit ויכול לגרום בעיות אם הcommit כבר נדחף לremote
12. אם הCommit עדיין לא נדחף לremote יותר מתאים git reset כי אין חשש לשנות היסטוריה כיוון שהוא קיים רק אצלנו אותו commit
13. לאחרי שcommit נדחף יכול להיות שמפתחים אחרים כבר משכו אותו עם git pull לכן אם נשתמש עם git reset אז ישתנה היסטוריית הcommit וזה יגרום לבלבול למי שכבר עובד על אותו מאגר אבל git revert משאיר את ההיסטוריה כמו שהיא ומוסיף commit חדש שמבטל את השינויים ולכן זה הפיתרון הכי טוב אם הCommit נמצא בremote.
